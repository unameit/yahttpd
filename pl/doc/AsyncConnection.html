Documentation

Klasa ta jest tylko interfejsem który muszą implementować obiekty chcące być obsługiwane przez ConnectionsManager'a.

Generalnie opiera się na mechanizmie asynchronicznych unix'owych socket'ów, ale ich obsługa odbywa się w sposób zdarzeniowy, tzn. gdy możliwy jest zapis/odczyt na danym deskryptorze, to generowane jest odpowiednie zdarzenie obsługujące tę sytuację (czyli wywoływana odpowiednia metoda dla obiektu klasy imlementującej ten interfejs).

virtual bool isReadNeeded() const
chcesz czytać?

virtual bool isWriteNeeded() const
chcesz pisać?

virtual void onReadable()
no to czytaj, bo teraz można

virtual void onWriteable()
analogicznie - pisz

virtual operator int() const
daj mi swój UNIXowy deskryptor pliku/gniazda. -1 oznacza, że połączenie jest już zamknięte.

virtual void finalize()
jeśli -1 == (int) *this, więc połączenie jest już zamknięte. wywołaj więc wtedy ten niby destruktor

virtual bool isFinalized() const
czy zostało już wywołane finalize() ? musisz zapamiętać ten fakt, żeby inne obiekty nie przeoczyły re-inicjalizacji połączenia (to zamiast niepotrzebnych new i delete)
